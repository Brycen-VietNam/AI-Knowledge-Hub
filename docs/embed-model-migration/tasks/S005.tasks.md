### T003: Eval harness CLI — recall@10 + MRR computation

**Status**: DONE

**File(s)**
- `backend/rag/eval/__init__.py` — create (empty, makes package importable)
- `backend/rag/eval/multilingual_recall.py` — create

**Change description**
CLI module runnable as `python -m backend.rag.eval.multilingual_recall --model <model_name>`.
Loads fixtures from `multilingual_recall.fixtures.json` (path relative to module).
For each query: calls `OllamaEmbedder.embed_query(query)` → issues pgvector cosine `<=>` search → retrieves top-10 doc IDs → checks if any `expected_doc_ids` appear in top-10.
Computes: recall@10 (hit rate) + MRR globally, per `query_lang`, per `category`.
Writes structured result dict to stdout (JSON) and passes to T004 report writer.
`--model` flag overrides `EMBEDDING_MODEL` env var (AC6).
No mxbai baseline computed (D09).

**Review criteria**
- [ ] Module importable as `python -m backend.rag.eval.multilingual_recall` (AC6)
- [ ] `--model` flag plumbed via `os.environ["EMBEDDING_MODEL"] = args.model` before embedder init (AC6)
- [ ] Recall@10 formula: `hits / total_queries` where hit = any expected_doc_id in top-10
- [ ] MRR formula: `mean(1/rank)` where rank is position of first hit (0 if no hit in top-10)
- [ ] Outputs metrics globally + per lang + per category
- [ ] Requires live Ollama + populated DB — test via `pytest.mark.integration` skip gate (no live Ollama in CI)
- [ ] Rule A001: harness imports `OllamaEmbedder` from `backend.rag.embedder` — no direct DB access; use existing retriever interface
- [ ] Rule P002: embedding calls batched (not per-query serial embed loop) — or clearly noted if sequential is required for eval correctness

**Test command**
```bash
pytest tests/rag/test_multilingual_recall.py -v -m "not integration"
```
_(integration smoke skipped in CI — runs manually with live Ollama + DB)_

**Parallel**: after:T001,T002
**Size estimate**: ~80 lines
