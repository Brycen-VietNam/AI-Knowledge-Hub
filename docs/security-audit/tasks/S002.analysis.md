## T003 — `_verify_local_jwt` tv claim check

### Code Map
```python
# backend/auth/dependencies.py — CURRENT STATE (relevant excerpt)

# Line 16: fastapi imports
from fastapi import Depends, Header, Request   ← HTTPException NOT imported ⚠️

async def _verify_local_jwt(token: str, db: AsyncSession) -> AuthenticatedUser | None:
    # L64: decode JWT
    payload = jwt.decode(token, _LOCAL_SECRET, algorithms=["HS256"],
                         options={"require": ["exp", "sub", "user_id"]})
    # L71: extract user_id
    user_id = uuid.UUID(payload["user_id"])

    # L76: DB lookup — CURRENT (returns only id)
    result = await db.execute(
        text("SELECT id FROM users WHERE id = :user_id AND is_active = TRUE")
        .bindparams(user_id=user_id)
    )
    row = result.fetchone()
    if row is None:
        return None        ← inactive user → None → OIDC fallthrough (correct)

    return AuthenticatedUser(
        user_id=user_id,
        user_group_ids=[],
        auth_type="oidc",  ← kept as "oidc" for local JWT (project decision T002)
    )
```

### ❌ SECURITY GAP — tv mismatch must NOT return None
If `tv` mismatch returns `None`, `verify_token` falls through to `verify_oidc_token` with a known local JWT → OIDC decode attempt with RS256/ES256 will fail, resulting in 401 eventually — **but** the error code will be OIDC-related, not `ERR_TOKEN_INVALIDATED`, and the flow is semantically wrong (we know this is a local token). Must raise directly.

### ⚠️ GAP — `HTTPException` not imported
`_errors.auth_error(request, ...)` requires a `Request` object — `_verify_local_jwt` has no `Request` param (and should not add one — would require signature change across callers).
Fix: import `HTTPException` from fastapi and raise directly.

```python
# Add to import line:
from fastapi import Depends, Header, HTTPException, Request
```

### ⚠️ Note — `request_id` in error detail
`_verify_local_jwt` has no `Request` param → no access to `request.state.request_id`.
Acceptable: use `str(uuid.uuid4())` inline for this edge case (consistent with auth_error fallback pattern in `_errors.py`).

### Recommended Change
```python
# After SELECT extension:
result = await db.execute(
    text("SELECT id, token_version FROM users WHERE id = :user_id AND is_active = TRUE")
    .bindparams(user_id=user_id)
)
row = result.fetchone()
if row is None:
    return None  # inactive → OIDC fallthrough (correct)

db_tv: int = int(row[1]) if row[1] is not None else 1
jwt_tv: int = int(payload.get("tv", 1))  # pre-S001 tokens have no 'tv' → default 1

if jwt_tv != db_tv:
    import uuid as _uuid
    raise HTTPException(
        status_code=401,
        detail={"error": {
            "code": "ERR_TOKEN_INVALIDATED",
            "message": "Session invalidated — please log in again",
            "request_id": str(_uuid.uuid4()),
        }},
    )

return AuthenticatedUser(user_id=user_id, user_group_ids=[], auth_type="oidc")
```
> `uuid` is already imported at module level — no need for `import uuid as _uuid`; just use `uuid.uuid4()`.

### Required Change (both paths)
Add `, token_version = token_version + 1` to the UPDATE SET clause:
```sql
-- Before:
UPDATE users SET password_hash = :hash, must_change_password = TRUE WHERE id = :user_id
-- After:
UPDATE users SET password_hash = :hash, must_change_password = TRUE,
token_version = token_version + 1 WHERE id = :user_id
```
No `RETURNING` needed (admin does not re-issue tokens for target user — contrast with D-SA-08).

### ✅ S001 Confirmed Safe
Both UPDATEs already use `text().bindparams()` with named params `:hash` and `:user_id` — no f-string interpolation. Adding `token_version = token_version + 1` is a static SQL fragment (no new user input injected).
