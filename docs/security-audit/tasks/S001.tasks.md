### T003: `auth.py` — new `POST /v1/auth/refresh` route

**Status**: TODO

**File(s)**
- `backend/api/routes/auth.py` — modify: add new route handler
- `tests/api/test_auth_routes.py` — modify: add refresh route tests (success + 4 failure paths)

**Change description**
Add `POST /v1/auth/refresh` endpoint. Accepts JSON body `{"refresh_token": "<token>"}`.
Calls `verify_refresh_token(token)` — on failure returns 401 `AUTH_TOKEN_INVALID`.
On success: looks up user (SELECT id, token_version, is_active, sub from users WHERE id=:uid AND is_active=TRUE).
Issues new access token via `create_access_token(sub, user_id, token_version)` and new refresh token.
Returns `{"access_token": ..., "refresh_token": ..., "token_type": "bearer", "expires_in": ...}`.
Rejects OIDC-format tokens with `AUTH_UNSUPPORTED` (D-SA-07) — detected by decode failure with `JWT_REFRESH_SECRET`.
No `verify_token` dependency — uses `verify_refresh_token` only (dedicated dependency per WARM D-SA).
Rate limit: 30 req/min per IP (half of query limit, lighter than login 10/min).

**Review criteria**
- [ ] Route uses `/v1/` prefix (R004)
- [ ] No `verify_token` dep — uses `verify_refresh_token` (C003: not exempt but different dependency)
- [ ] OIDC tokens → 401 `AUTH_UNSUPPORTED` (D-SA-07) — not `AUTH_FAILED`
- [ ] Both new access token AND new refresh token issued on success (token rotation)
- [ ] Rate limiter applied
- [ ] Error shape matches A005: `{"error": {"code", "message", "request_id"}}`
- [ ] Tests cover: success, expired refresh token, tampered token, OIDC token rejected, inactive user

**Test command**
```bash
pytest tests/api/test_auth_routes.py -v -k "refresh"
```

**Parallel**: after:T001
**Size estimate**: ~55 lines (auth.py new handler) + ~60 lines (tests)
