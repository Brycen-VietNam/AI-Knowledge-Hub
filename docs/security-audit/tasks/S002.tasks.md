### T003: `_verify_local_jwt` — extend SELECT + `tv` claim mismatch → 401

**Status**: TODO

**File(s)**
- `backend/auth/dependencies.py` — modify: extend SELECT + tv validation logic
- `tests/auth/test_dependencies.py` — modify: add tv mismatch test cases

**Change description**
In `_verify_local_jwt(token, db)`, make two changes:

**1. Extend SELECT (SI-02):**
```python
# Before:
text("SELECT id FROM users WHERE id = :user_id AND is_active = TRUE")
# After:
text("SELECT id, token_version FROM users WHERE id = :user_id AND is_active = TRUE")
```

**2. Add `tv` claim validation after successful DB lookup:**
```python
row = result.fetchone()
if row is None:
    return None

db_token_version: int = int(row[1]) if row[1] is not None else 1
jwt_tv: int = int(payload.get("tv", 1))

if jwt_tv != db_token_version:
    # Token has been invalidated (admin reset bumped token_version).
    # Raise 401 directly — do NOT return None (avoids silent OIDC fallthrough for local tokens).
    from fastapi import HTTPException
    raise HTTPException(
        status_code=401,
        detail={"error": {"code": "ERR_TOKEN_INVALIDATED",
                          "message": "Session invalidated — please log in again",
                          "request_id": ""}},
    )

return AuthenticatedUser(
    user_id=user_id,
    ...
)
```

**Important**: Raise `HTTPException` (not return None) on tv mismatch.
Returning None would cause `verify_token` to try the OIDC path on a known local token — this is a security gap. Once we've confirmed the token is a valid local JWT, tv mismatch must terminate with 401.

**Note on `request_id`**: `_verify_local_jwt` has no `Request` param. Using empty string for `request_id` is acceptable here; callers with full `Request` context already generate ids for other 401 paths.

**Review criteria**
- [ ] SELECT returns `(id, token_version)` — not just `(id,)` (SI-02)
- [ ] `jwt_tv` read from payload (defaults to 1 if `tv` absent — pre-S001 tokens safe)
- [ ] Mismatch → `HTTPException 401 ERR_TOKEN_INVALIDATED` (not `return None`)
- [ ] Match → existing AuthenticatedUser construction unchanged
- [ ] OIDC tokens (no `tv` claim) already fail `_pyjwt.decode` with `_LOCAL_SECRET` → fall through naturally (not affected)
- [ ] Test: matching tv → 200 (existing behavior preserved)
- [ ] Test: stale tv (jwt=1, db=2) → 401 ERR_TOKEN_INVALIDATED
- [ ] Test: missing `tv` claim (pre-S001 tokens) → treated as tv=1 → passes if db=1
- [ ] Test: inactive user still → None (return None path unaffected)
