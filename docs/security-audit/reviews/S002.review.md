#### ⚠️ WARNING — Non-blocking
- **`auth_type="oidc"` for local JWT** ([backend/auth/dependencies.py:109](backend/auth/dependencies.py#L109)) is a pre-existing quirk carried forward — labeling a locally-issued HS256 token as "oidc" may confuse audit trails. Out of scope for S002 (already noted in T002 decision), but worth a follow-up.
- **`request_id` source** in the 401 payload is a fresh `uuid.uuid4()` rather than the caller's request id. `_verify_local_jwt` has no `Request` param, so this is the pragmatic choice; consider threading `request` into the helper in a later pass so all 401s share the caller's id.

### Suggested follow-up test

async def test_tv_mismatch_does_not_leak_to_oidc_path(client, local_jwt_stale_tv):
