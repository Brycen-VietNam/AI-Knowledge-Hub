### T002: Sliding window chunker with CJK-aware token counting

**Status**: TODO

**File(s)**
- `backend/rag/chunker.py` — action: modify (add `chunk_document` function)
- `tests/rag/test_chunker.py` — action: modify (add chunking tests)

**Change description**
Implement `chunk_document(content, lang, doc_id) -> list[Chunk]` using sliding window:
- `CHUNK_SIZE = int(os.getenv("CHUNK_SIZE", "512"))`
- `CHUNK_OVERLAP = int(os.getenv("CHUNK_OVERLAP", "50"))`
- Token counting: CJK langs (ja/ko/zh/vi) → `len(TokenizerFactory.get(lang).tokenize(text))`; latin → `len(text.split())`
- Split on word boundaries approximation; advance by `(CHUNK_SIZE - CHUNK_OVERLAP)` tokens per step.

**Review criteria**
- [ ] `CHUNK_SIZE` and `CHUNK_OVERLAP` read from env vars with defaults
- [ ] CJK token counting uses `TokenizerFactory.get(lang).tokenize(text)` (R005)
- [ ] Latin uses `text.split()` estimate
- [ ] `chunk_index` is 0-based and sequential
- [ ] Test: english text → chunks ≤ CHUNK_SIZE tokens
- [ ] Test: japanese text → CJK tokenizer used (mock `TokenizerFactory`)
- [ ] Test: overlap between consecutive chunks ≈ CHUNK_OVERLAP tokens
- [ ] Rule satisfied: R005, C005

**Test command**
