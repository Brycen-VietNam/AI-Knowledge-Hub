### T002: verify_api_key implementation

**Status**: DONE

**File(s)**
- `backend/auth/api_key.py` — action: create

**Change description**
Implement `verify_api_key(request: Request, db: AsyncSession) -> AuthenticatedUser`.
Logic: extract `X-API-Key` header → SHA-256 hash (stdlib `hashlib`) → ORM SELECT with
`WHERE key_hash=:hash AND is_active=TRUE` → if not found raise 401 → UPDATE
`last_used_at=now()` → return `AuthenticatedUser(auth_type="api_key")`.
Import `AuthenticatedUser` from `backend.auth.types` (created in S004 — use
`TYPE_CHECKING` guard or string annotation `"AuthenticatedUser"` until S004 exists).

**Review criteria**
- [ ] `verify_api_key` signature: `async def verify_api_key(request: Request, db: AsyncSession)`
- [ ] `X-API-Key` header missing or empty → raises `auth_error(..., "AUTH_MISSING", ..., 401)`
- [ ] Hash: `hashlib.sha256(key.encode()).hexdigest()` — stdlib only, no new dep

- [ ] Not found → raises `auth_error(..., "AUTH_INVALID_KEY", ..., 401)` (no detail about why)
- [ ] On success: `UPDATE api_keys SET last_used_at=now() WHERE id=:id` (via ORM or text())
