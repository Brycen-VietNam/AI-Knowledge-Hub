## Patterns to Follow

- **AuthenticatedUser construction**: `AuthenticatedUser(user_id=..., user_group_ids=..., auth_type="api_key")` — matches existing callers in api_key.py L56 and oidc.py L128
- **auth_error pattern**: `raise auth_error(request, "CODE", "message", 401)` — used in api_key.py L34, L47 and oidc.py L115
- **Header extraction in FastAPI**: `request.headers.get("X-API-Key", "")` in api_key.py — T002 uses FastAPI `Header()` dependency injection instead (task spec)
- **Frozen dataclass**: tasks spec requires `@dataclass(frozen=True)` + `Literal["api_key", "oidc"]` — upgrade from current mutable `@dataclass` + plain `str`
- **Spec header comment pattern**: every file starts with `# Spec: ...`, `# Task: ...`, `# Decision: ...`, `# Rule: ...`

### T002 — backend/auth/dependencies.py (create, ~35 lines)
```python
async def get_db() -> AsyncGenerator[AsyncSession, None]:
    # Guard: async_session_factory is None in test env without DATABASE_URL
    async with async_session_factory() as session:
        yield session

async def verify_token(
    request: Request,
    x_api_key: str | None = Header(default=None, alias="X-API-Key"),
    authorization: str | None = Header(default=None),
    db: AsyncSession = Depends(get_db),
) -> AuthenticatedUser:
    if x_api_key:                          # D05: API-key takes precedence
        return await verify_api_key(request, db)
    if authorization and authorization.startswith("Bearer "):
        token = authorization.removeprefix("Bearer ")
        return await verify_oidc_token(request, token, db)
    raise auth_error(request, "AUTH_MISSING", "Authentication required", 401)
```

Note: `verify_api_key` already extracts key from `request.headers` internally — T002 passes `request` and `db` through.
