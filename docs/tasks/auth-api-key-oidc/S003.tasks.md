**Change description**
Add three functions to `oidc.py`:
1. `verify_oidc_token(request, token, db)` — full JWT validation pipeline.
   All exceptions caught broadly → re-raise as `AUTH_TOKEN_INVALID` (no token content leaked).
2. `_jit_upsert_user(db, sub, email, display_name)` — parameterized `text()` INSERT ON CONFLICT.
3. `_resolve_group_ids(db, group_names)` — ORM `UserGroup.name.in_(group_names)` SELECT.

**Review criteria**
- [ ] `jwt.decode` called with `algorithms=["RS256","ES256"]`, `audience=OIDC_AUDIENCE`, `issuer=OIDC_ISSUER`, `options={"require": ["exp","iss","aud","sub"]}`
- [ ] Entire try/except block catches `Exception` → raises `auth_error(..., "AUTH_TOKEN_INVALID", ..., 401)`
- [ ] No JWT payload fields in exception detail (AC4 — no token content exposed)
- [ ] `groups` claim: `payload.get(OIDC_GROUPS_CLAIM) or []` — empty/absent → `[]` (D06)
- [ ] JIT UPSERT uses `text()` with named params `:sub, :email, :display_name` — no f-string (S001)
- [ ] ON CONFLICT DO UPDATE updates `email` and `display_name` on each login
- [ ] Groups resolution: `select(UserGroup.id).where(UserGroup.name.in_(group_names))` — parameterized
- [ ] Returns `AuthenticatedUser(user_id=..., user_group_ids=..., auth_type="oidc")`
