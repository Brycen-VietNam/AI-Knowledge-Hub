### T002: verify_token + get_db in dependencies.py

**Status**: TODO

**File(s)**
- `backend/auth/dependencies.py` — action: create

**Change description**
Create `dependencies.py` with two functions:
1. `get_db()` — async generator yielding `AsyncSession` from `async_session_factory`
   (imported from `backend.db.session`).
2. `verify_token(request, x_api_key, authorization, db)` — dispatch logic:
   `X-API-Key` present → `verify_api_key`; `Authorization: Bearer` → `verify_oidc_token`;
   neither → `AUTH_MISSING`. X-API-Key takes precedence when both headers present (D05).

**Review criteria**
- [ ] `verify_token` signature uses FastAPI `Header()` for `x_api_key` (alias `"X-API-Key"`) and `authorization`
- [ ] `x_api_key` check is FIRST (D05 — API-key takes precedence)
- [ ] Bearer extraction: `authorization.removeprefix("Bearer ")` (Python 3.9+) or `authorization[7:]`
- [ ] Neither header → `auth_error(request, "AUTH_MISSING", ..., 401)`
- [ ] `get_db` uses `async with async_session_factory() as session: yield session`
- [ ] Imports `async_session_factory` from `backend.db.session`
- [ ] Imports `verify_api_key` from `backend.auth.api_key` and `verify_oidc_token` from `backend.auth.oidc`
- [ ] Imports `AuthenticatedUser` from `backend.auth.types`
- [ ] Rule satisfied: A002 (no reverse deps — auth imports from db only, not api/rag)
- [ ] Rule satisfied: A005 (AUTH_MISSING error shape via _errors.py)
