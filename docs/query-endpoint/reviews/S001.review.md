### Issues Found

#### ❌ BLOCKER — Must resolve before APPROVED

**B01 — R007 timeout violation**
File: `backend/api/routes/query.py` line ~110
`timeout=1.0` deviates from HARD rule R007 which mandates `timeout=1.8` (1800ms SLA).
The 1.0s value matches the A2 confirmed assumption (retrieval budget = 1.0s, LLM budget = 0.8s),
and the 504 message already reads "1000ms SLA" consistently. This is an intentional budget split —
but R007 must be reconciled: either document the deviation in WARM memory or update the R007 comment
in the route header to reference the A2 decision.

**Resolution options:**
1. Add comment `# R007: 1.8s total = 1.0s retrieval (A2) + 0.8s LLM (S002-T002)` to the route header
2. Extract constant: `RETRIEVAL_TIMEOUT = 1.0  # A2: retrieval budget within 1.8s SLA`

**W02 — No generic exception handler for search() failures**

Extract to module constant or env-configurable setting consistent with A004 pattern.
