## Issues Found

### ⚠️ WARN (non-blocking) — `audit_logs.doc_id` has no ON DELETE behavior

`audit_logs.doc_id REFERENCES documents(id)` has no `ON DELETE` clause (defaults to RESTRICT).

This means: deleting a document will fail if any audit log references it.

**Options**:
1. `ON DELETE CASCADE` — delete audit logs with document (loses audit trail)
2. `ON DELETE SET NULL` — keep audit record, nullify doc_id (requires `doc_id` nullable)
3. **Leave as RESTRICT** (current) — force app layer to archive/purge before document delete

**Recommendation**: Keep RESTRICT (current behavior) — preserves audit trail integrity. Document deletion should be handled by app layer explicitly. Add a comment.

> This is a design choice, not a bug. Flagging for awareness only.
