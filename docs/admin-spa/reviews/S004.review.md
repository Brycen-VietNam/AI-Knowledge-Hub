## Issues Found

### ⚠️ WARNING — Should fix

**1. No request logging on `/v1/metrics`**
The endpoint at [backend/api/routes/admin.py:439-486](backend/api/routes/admin.py#L439) performs 4 separate DB queries but emits no log line. Per the Full check requirement (request_id in all log entries), this is a gap. It does not violate R006 (which only applies to document access), but observability is missing.
```python
# Add near top of get_metrics():
import logging
logger = logging.getLogger(__name__)
logger.info("metrics_request", extra={"request_id": str(uuid.uuid4()), "user_id": str(user.user_id)})
```

**2. `admin_assign_user_groups` has N+1 insert loop (P004 risk)**
File: [backend/api/routes/admin.py:402-409](backend/api/routes/admin.py#L402)
```python
for group_id in body.group_ids:
    await db.execute(...)  # one query per group_id
```
Not a hard blocker since `group_ids` is expected to be small (< 10), and `ON CONFLICT DO NOTHING` is correct. But it violates the spirit of P004 for larger payloads. Should use a bulk insert with `VALUES` tuples. Tag as tech-debt.

**3. `health-badge--ok/error` CSS references undefined `--success`/`--danger` variables**
File: [frontend/admin-spa/src/index.css:1249-1250](frontend/admin-spa/src/index.css#L1249)
```css
.health-badge--ok .health-dot { background: var(--success); }
.health-badge--error .health-dot { background: var(--danger); }
```
The `:root` token block defines `--emerald` and `--red` but NOT `--success` or `--danger`. These CSS vars will silently fall back to `initial` (transparent dot). The health dots will be invisible in production.
```css
/* Fix: use defined tokens */
.health-badge--ok .health-dot { background: var(--emerald); }
.health-badge--error .health-dot { background: var(--red); }
```

**4. DashboardPage test count mismatch — test 5 missing**
The task spec requires 6 DashboardPage tests including: "`useAutoRefresh` is called with 60_000 interval (mock hook)". The actual test file has 6 tests but test 5 (`renders QueryVolumeChart wrapper`) replaces the `useAutoRefresh` interval verification. The hook's 60s wiring is not explicitly tested at the DashboardPage integration level. Minor gap — the hook itself is tested in isolation with correct interval.
