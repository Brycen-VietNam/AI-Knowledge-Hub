## Issues Found

### ⚠️ WARNING — Should fix

**1. Double `listUsers` call on initial mount (UsersTab.tsx L35–43)**

The component fires two `listUsers('')` calls on mount: one explicit in the `useEffect([loadUsers])` at L37, and one from the debounce `useEffect([search, loadUsers])` at L42 (which also fires immediately with `search=''`).

File: [frontend/admin-spa/src/components/UsersTab.tsx](frontend/admin-spa/src/components/UsersTab.tsx#L35-L43)

Fix: Remove the explicit `loadUsers('')` from the initial effect, relying solely on the debounce effect:
```typescript
// Initial load: groups only; debounce effect handles users
useEffect(() => {
  listGroups().then(setAllGroups)
}, [])
```
The debounce effect runs on mount with `search=''` and covers the initial fetch.

**2. Wrong i18n key used for toggle-active error toast (UsersTab.tsx L57)**

`handleToggleActive` sets the error toast message to `t('col_active')` — the column header — instead of a meaningful error key. There is no dedicated i18n key for this failure in the spec, but using the column header as an error message is incorrect.

File: [frontend/admin-spa/src/components/UsersTab.tsx](frontend/admin-spa/src/components/UsersTab.tsx#L57)

Fix: Either reuse a generic error key or add `toggle_active_error` to i18n. Minimal fix using existing keys:
```typescript
setToast({ msg: t('group_save_error'), type: 'error' })  // reuse closest generic
```
Better: add `"toggle_active_error": "Failed to update user status"` to all 4 locales.

**3. `loadGroups` error in UsersTab silently ignored (UsersTab.tsx L36)**

`listGroups().then(setAllGroups)` has no `.catch()`. If the groups fetch fails on mount, `allGroups` stays `[]` and the AssignGroupModal opens with no checkboxes — no user feedback.

File: [frontend/admin-spa/src/components/UsersTab.tsx](frontend/admin-spa/src/components/UsersTab.tsx#L36)

Fix:
```typescript
listGroups().then(setAllGroups).catch(() => {
  setToast({ msg: t('group_delete_error'), type: 'error' })
})
```
