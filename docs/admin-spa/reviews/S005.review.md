### ⚠️ WARNING — nginx.conf: security headers on server block only

Security headers (`X-Frame-Options`, `X-Content-Type-Options`, `Referrer-Policy`) are on the `server {}` block but NOT inside the `location` blocks. In nginx, `add_header` directives in parent blocks are **not inherited** when a child `location` block also has `add_header`. Since the cache `location` block adds `Cache-Control`, it will NOT inherit the security headers.

**File:** [frontend/admin-spa/nginx.conf:19-23](frontend/admin-spa/nginx.conf#L19-L23)

**Fix:** Move security headers inside each `location` block, or use `always` parameter and restructure:
```nginx
location ~* \.(js|css|png|...)$ {
    expires 1y;
    add_header Cache-Control "public, immutable";
    add_header X-Frame-Options "SAMEORIGIN";
    add_header X-Content-Type-Options "nosniff";
    add_header Referrer-Policy "strict-origin-when-cross-origin";
}
```
Note: This is a known nginx behavior. The frontend-spa `nginx.conf` has the same pattern — recommend fixing both together.
