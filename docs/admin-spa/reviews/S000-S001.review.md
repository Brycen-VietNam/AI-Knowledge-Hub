### The One Flag — admin.py:252–255

```python
set_clause = ", ".join(updates)  # safe: only hardcoded column names, no user values
stmt = text(
    f"UPDATE user_groups SET {set_clause} WHERE id = :group_id "
    "RETURNING id, name, is_admin, created_at"
).bindparams(**params)
```

**Assessment: ⚠️ WARNING, not a blocker.**

`updates` contains only two possible hardcoded strings (`"name = :name"`, `"is_admin = :is_admin"`). Neither contains user input. User values reach the DB only via `.bindparams(**params)`. Technically safe.

However, the f-string pattern is a code smell. A future maintainer could extend `updates` with a user-supplied value.

**Suggested fix — explicit static SQL strings:**

```python
# Two fields → four possible states; enumerate them explicitly
if body.name is not None and body.is_admin is not None:
    sql = "UPDATE user_groups SET name=:name, is_admin=:is_admin WHERE id=:group_id RETURNING id, name, is_admin, created_at"
elif body.name is not None:
    sql = "UPDATE user_groups SET name=:name WHERE id=:group_id RETURNING id, name, is_admin, created_at"
else:
    sql = "UPDATE user_groups SET is_admin=:is_admin WHERE id=:group_id RETURNING id, name, is_admin, created_at"
stmt = text(sql).bindparams(**params)
```
