## Critical Corrections for Implementation

### C1 ÔÇö `GroupItem.member_count` vs `user_count`
Backend returns `user_count` (not `member_count`).
```typescript
// CORRECT interface:
export interface GroupItem {
  id: number; name: string; is_admin: boolean;
  user_count: number;   // NOT member_count
  created_at: string;
}
```

### C2 ÔÇö `assignGroups` API semantics = ADDITIVE, not REPLACE
`POST /v1/admin/users/{user_id}/groups` adds groups (upsert). Does NOT remove existing.
To fully replace groups: compare current vs desired Ôćĺ delete removed, insert added.
AssignGroupModal must compute delta and call both `POST groups` + `DELETE groups/{id}` per removal.

```typescript
// assignGroups implementation (T001):
export async function assignGroups(userId: string, currentGroupIds: number[], newGroupIds: number[]): Promise<void> {
  const toAdd = newGroupIds.filter(id => !currentGroupIds.includes(id))
  const toRemove = currentGroupIds.filter(id => !newGroupIds.includes(id))
  if (toAdd.length > 0) {
    await apiClient.post(`/v1/admin/users/${userId}/groups`, { group_ids: toAdd })
  }
  for (const gid of toRemove) {
    await apiClient.delete(`/v1/admin/users/${userId}/groups/${gid}`)
  }
}
```
**Signature update:** `assignGroups(userId, currentGroupIds, newGroupIds)` ÔÇö needs current state.
AssignGroupModal must pass `user.groups.map(g => g.id)` as `currentGroupIds`.

### C3 ÔÇö No server-side search on `GET /v1/admin/users`
Backend has no `search` query param. Frontend must filter client-side:
```typescript
// listUsers ÔÇö fetch all, filter locally:
export async function listUsers(): Promise<UserItem[]> {
  const res = await apiClient.get('/v1/admin/users')
  return res.data.items
}
// UsersTab: const filtered = users.filter(u => u.email.includes(search) || u.sub.includes(search))
```
Remove `search?` param from `listUsers` signature. Debounce still applies (now for local filter trigger).

### C4 ÔÇö i18n namespace must be `"usersGroups"` (not flat keys)
All S003 i18n keys Ôćĺ `{ "usersGroups": { "tab_groups": "Groups", ... } }`.
Component usage: `t('usersGroups.tab_groups')`.
Matches existing S002 pattern: `t('documents.page_title')`.

### C5 ÔÇö `updateGroup` body shape
Backend `PUT /v1/admin/groups/{id}` body: `{ name?, is_admin? }` (partial).
GroupFormModal always sends both Ôćĺ `{ name: string, is_admin: boolean }`. Fine.

### C6 ÔÇö `UserItem` interface (add `sub`, `display_name` fields)
Backend returns `sub` and `display_name`. Update interface:
```typescript
export interface UserItem {
  id: string; sub: string; email: string;
  display_name: string | null; is_active: boolean;
  groups: { id: number; name: string; is_admin: boolean }[];
```
