## Conflicts / Gaps Found

### ⚠️ WARNING — Module-level env var validation
T002 spec says "check OIDC_ISSUER, OIDC_AUDIENCE, OIDC_JWKS_URI env vars at import time; raise RuntimeError if any missing."
This means `import backend.auth.oidc` will **fail** in test environments unless env vars are set.
→ **Tests must use `monkeypatch.setenv` BEFORE import**, or use `importlib.reload()`.
→ Consider: validate at first use (lazy) instead of import time? Task spec says import-time — follow spec.

### ⚠️ WARNING — AuthenticatedUser duplication
`api_key.py` defines `AuthenticatedUser` as a local dataclass. `oidc.py` needs the same type.
Options: (a) import from `api_key.py`, (b) duplicate in `oidc.py`, (c) extract to shared module now.
→ **Recommended**: Import `AuthenticatedUser` from `backend.auth.api_key` (avoid duplication). S004 will move it to `types.py` and update both imports.

### ⚠️ WARNING — `_jit_upsert_user` uses raw `text()` SQL
Task spec requires `text()` INSERT ON CONFLICT with named params. This is correct for UPSERT (SQLAlchemy ORM doesn't natively support ON CONFLICT DO UPDATE cleanly).
→ Must use `text().bindparams()` — NOT f-string. S001 rule.
→ PostgreSQL-specific syntax: `INSERT INTO users ... ON CONFLICT (sub) DO UPDATE SET ...`

## Key Design Points

### JWKS Cache Strategy
- Module-level `_jwks_cache: dict[str, RSAPublicKey]` keyed by `kid`
- `_jwks_fetched_at: float` tracks last fetch time (via `time.monotonic()`)
- On `kid` miss or TTL expired → `_refresh_jwks_cache()` via httpx
- httpx timeout = 2.0s (task spec)
- TTL default = 3600s from `OIDC_JWKS_CACHE_TTL` env var

### JWT Validation (S002 rule compliance)
- `jwt.decode()` with: `algorithms=["RS256", "ES256"]`, `audience`, `issuer`, `options={"require": ["exp", "iss", "aud", "sub"]}`
- Entire decode block wrapped in `try/except Exception` → `AUTH_TOKEN_INVALID` 401
- NO token payload fields in error detail (AC4)

### JIT User Provisioning (D02)
- `INSERT INTO users (id, sub, email, display_name) VALUES (:id, :sub, :email, :display_name) ON CONFLICT (sub) DO UPDATE SET email = :email, display_name = :display_name`
- Returns `user.id` via `RETURNING id`

### Groups Resolution (D01, D06)
- `payload.get(OIDC_GROUPS_CLAIM) or []` — empty/absent is OK (permissive, D06)
- `select(UserGroup.id).where(UserGroup.name.in_(group_names))` → list[int]
- Unknown group names silently ignored (no error)
