## Patterns to Follow

### Migration style (from `001_create_core_schema.sql`)
- Header block: `-- Migration: NNN_name.sql`, `-- Spec:`, `-- Task:`, `-- Requires:`
- Section markers: `-- FORWARD` / `-- ROLLBACK`
- `ROLLBACK` section at bottom, commented, **reverse order**
- `CREATE TABLE IF NOT EXISTS` pattern
- Inline comments on columns (e.g. `-- placeholder; FK added by...`)

### ORM model style (from `audit_log.py`)
- Spec/Task header comment block at top
- `import uuid` + `from datetime import datetime`
- `from sqlalchemy import ForeignKey, func` + `from sqlalchemy.orm import Mapped, mapped_column`
- `from .base import Base`
- No ORM relationships, pure column definitions only

### Test style (from `test_models.py`)
- `scope="module"` engine fixture with SQLite in-memory
- Per-test `session` fixture with `s.rollback()` teardown
- `inspect(engine)` for structural assertions (columns, FKs)
- Direct insert + flush for roundtrip tests
