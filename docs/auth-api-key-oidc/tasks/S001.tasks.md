### T001: Migration 004 — users + api_keys tables + audit_logs FK

**Status**: REVIEWED

**File(s)**
- `backend/db/migrations/004_create_users_api_keys.sql` — action: create

**Change description**
Create `users` and `api_keys` tables, add `idx_users_sub` index, then migrate
`audit_logs.user_id` from TEXT to UUID FK. `audit_logs` is empty (D08) so no
UPDATE step needed. FK added with `NOT VALID` + `VALIDATE` pattern to avoid lock.
Migration ends with commented ROLLBACK section (reverse order).

**Review criteria**
- [ ] `users` table: UUID PK via `gen_random_uuid()`, `sub TEXT UNIQUE NOT NULL`, `email`/`display_name` nullable, `is_active BOOL DEFAULT TRUE`, `created_at TIMESTAMPTZ DEFAULT now()`
- [ ] `api_keys` table: UUID PK, `user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE`, `key_hash TEXT NOT NULL UNIQUE`, `user_group_ids INTEGER[] NOT NULL DEFAULT '{}'`, `last_used_at TIMESTAMPTZ` nullable
- [ ] `CREATE INDEX idx_users_sub ON users(sub)` present
- [ ] `ALTER TABLE audit_logs ALTER COLUMN user_id TYPE UUID USING user_id::uuid` before FK add
- [ ] FK added as `NOT VALID` then `VALIDATE CONSTRAINT` (two statements)
- [ ] Rollback section present (commented, reverse order)
- [ ] Header: `-- Requires: migrations 001, 002, 003 applied`
- [ ] Rule satisfied: A006 (numbered migration with rollback)
- [ ] Rule satisfied: S001 (no string interpolation — pure DDL, no dynamic SQL)

**Test command**
```bash

psql $DATABASE_URL -c "\d users" && \
```

**Rollback**
