Create numbered migration file following ARCH.md A006 pattern (header comment, FORWARD section, ROLLBACK section).
FORWARD: two `ALTER COLUMN DROP NOT NULL` statements + two `CREATE INDEX` partial indexes.
ROLLBACK: reverse order — drop indexes, UPDATE NULLs to safe default, restore NOT NULL constraints.
No data migration required — `DROP NOT NULL` does not touch existing rows.

**SQL to write**

FORWARD:
```sql

ALTER TABLE embeddings ALTER COLUMN user_group_id DROP NOT NULL;
CREATE INDEX idx_documents_public ON documents(id) WHERE user_group_id IS NULL;
CREATE INDEX idx_embeddings_public ON embeddings(doc_id) WHERE user_group_id IS NULL;
```

ROLLBACK:
```sql
-- DROP INDEX IF EXISTS idx_embeddings_public;
-- DROP INDEX IF EXISTS idx_documents_public;
-- UPDATE embeddings SET user_group_id = (SELECT MIN(id) FROM user_groups) WHERE user_group_id IS NULL;
-- UPDATE documents SET user_group_id = (SELECT MIN(id) FROM user_groups) WHERE user_group_id IS NULL;
-- ALTER TABLE embeddings ALTER COLUMN user_group_id SET NOT NULL;
-- ALTER TABLE documents ALTER COLUMN user_group_id SET NOT NULL;
```

**Review criteria**
- [ ] File named `005_nullable_user_group_id.sql` (sequential after 004)
- [ ] Header comment block: migration name, spec reference, requires: migrations 001–004
- [ ] FORWARD has exactly 4 statements: 2 × ALTER + 2 × CREATE INDEX
- [ ] `idx_embeddings_public` indexes `doc_id` (not `id`) — `doc_id` is the useful lookup field (clarify Q7)
- [ ] ROLLBACK uses `(SELECT MIN(id) FROM user_groups)` not hardcoded `1`
- [ ] ROLLBACK updates NULLs BEFORE restoring NOT NULL (order matters — constraint fails otherwise)

- [ ] Rule satisfied: ARCH.md A006 (numbered migration + rollback section)
