## Conflicts / Gaps Found

No ❌ security or rule violations detected.

⚠️ One gotcha: `ALTER COLUMN DROP NOT NULL` does **not** support `IF NOT EXISTS` in PostgreSQL 17.
Migration is not idempotent at the ALTER level — running twice will error with "column is already nullable".
Mitigation: document in migration header comment. Migration tooling (Alembic/Flyway) tracks applied versions. If running raw, wrap in a DO block:
```sql
DO $$
BEGIN
  IF EXISTS (
    SELECT 1 FROM information_schema.columns
    WHERE table_name='documents' AND column_name='user_group_id' AND is_nullable='NO'
  ) THEN
    ALTER TABLE documents ALTER COLUMN user_group_id DROP NOT NULL;
  END IF;
END$$;
```
Decision: **do not add the DO block** — project uses sequential numbered migrations with no re-run (ARCH.md A006). Keep it simple; document the non-idempotency in header.

⚠️ ROLLBACK order: `ALTER TABLE SET NOT NULL` must come AFTER the UPDATE that fills NULLs.
Wrong order = constraint violation. Tasks file already captures this — confirm in review criteria.
