## Notes for /implement

- `tests/db/test_models.py` uses **SQLite in-memory** — SQLite supports nullable columns without issue. The inspector `nullable` check works correctly with SQLite for column definitions set via SQLAlchemy `nullable=True/False`.
- SQLite does NOT enforce FK constraints by default — the NULL `user_group_id` on `Document` (which has `ForeignKey("user_groups.id")`) will insert fine in SQLite without a matching UserGroup row. This is acceptable for unit test scope; integration FK validation is PostgreSQL-level.
- `Mapped[Optional[int]]` in SQLAlchemy 2.x: `Optional[int]` = `int | None` — both are valid; prefer `Optional[int]` for Python 3.9 compatibility.
- `from typing import Optional` must be added to both model files — do NOT use `int | None` union syntax at top-level Mapped (SQLAlchemy 2.x handles both, but `Optional[int]` matches existing model style in `content_fts: Mapped[str | None]` in document.py — actually `str | None` is already used there, so `int | None` is also acceptable — follow existing file style).
- Scope boundary: S004 is ONLY ORM model nullability. Do NOT modify session.py, __init__.py, or migrations. Migration 005 (S001) is already done.
