```python
import statistics, time

class TestPerformance:
    """Integration: latency SLA under realistic data volume. AC10."""

    @pytest.mark.performance
    @pytest.mark.integration
    @pytest.mark.asyncio
    async def test_latency_p95_under_1800ms(self, large_seeded_session):
        """AC10: p95 < 1800ms with 10k embeddings, user has access to 1k."""
        latencies = []
        for _ in range(20):
            start = time.monotonic()
            await retrieve(
                query_embedding=[0.1] * 1024,
                user_group_ids=[1],
                top_k=10,
                session=large_seeded_session,
            )
            latencies.append((time.monotonic() - start) * 1000)
        latencies.sort()
        p95 = latencies[int(len(latencies) * 0.95)]
        assert p95 < 1800, f"p95 latency {p95:.0f}ms exceeds 1800ms SLA"
```

## Notes for /implement

- **Integration vs unit**: S002 tests (existing) use `AsyncMock` — unit scope. S003 tests use `seeded_session` — integration scope. Both live in `tests/rag/test_retriever_rbac.py`; mark integration tests with `@pytest.mark.integration` so they can be run separately.
- **`TEST_DATABASE_URL` required**: integration tests skip if env var not set — add `pytest.importorskip` or `pytest.mark.skipif(not os.getenv("TEST_DATABASE_URL"), ...)` at class level.
- **Fixture isolation**: `seeded_session` should use `SAVEPOINT` / nested transactions so each test gets a clean slate without full rollback cost.
- **pgvector seeding**: embeddings need real vector values — use `[float(i)/1024 for i in range(1024)]` pattern; distance ordering will be deterministic for tests.
- **BM25 seeding**: for `TestHybridRBAC`, seed `documents.content_fts` using `to_tsvector('simple', 'knowledge base test')` in the INSERT.
- **`large_seeded_session`**: use `executemany` or SQLAlchemy bulk insert — not 10k individual inserts.
- Import path assumption: tests run from repo root with `PYTHONPATH=.` set.
- `pytest.ini` or `pyproject.toml` must register custom marks: `integration`, `performance` (to avoid PytestUnknownMarkWarning).
