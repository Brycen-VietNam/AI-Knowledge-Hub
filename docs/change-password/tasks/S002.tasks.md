### T001: Add POST /v1/admin/users/{id}/password-reset route

**Status**: DONE

**File(s)**
- `backend/api/routes/admin.py` — modify: append new route

**Change description**
Append `POST /v1/admin/users/{id}/password-reset` to the existing admin users router.
Request body: `PasswordResetRequest` Pydantic model with discriminated union:
- `{"generate": true}` → call `generate_password(16)` → hash → store → return `{"password": plaintext}` 200
- `{"new_password": "..."}` → validate `8 ≤ len ≤ 128` → hash → store → return 204

Both branches:
- `Depends(require_admin)` (already pattern in file)
- OIDC guard: `if user.password_hash is None → 400 ERR_PASSWORD_NOT_APPLICABLE` (D3)
- 404: `if user not found → 404 ERR_USER_NOT_FOUND`
- Always set `must_change_password = True` on target user (D-CP-03 / Q3)
- Audit log: `admin_user_id`, `target_user_id`, `action="admin_password_reset"`, `timestamp`

**Review criteria**
- [ ] Route path: `/v1/admin/users/{id}/password-reset` (R004)
- [ ] `Depends(require_admin)` present (R003)
- [ ] `must_change_password = True` always set (Q3 resolved)
- [ ] OIDC users → 400 (D3)
- [ ] User not found → 404 (A005 error shape)
- [ ] `generate=true` → 200 + `{"password": ...}`; explicit new_password → 204 (D2)
- [ ] Audit log written (R006)
- [ ] Password length validated `8–128` for manual branch (S003)
- [ ] `generate_password()` imported from `backend/auth/utils.py` (S001/T003)
