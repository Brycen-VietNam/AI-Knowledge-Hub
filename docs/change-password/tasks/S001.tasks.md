### T004: Create users.py — PATCH /v1/users/me/password endpoint

**Status**: TODO

**File(s)**
- `backend/api/routes/users.py` — create

**Change description**
New route file. Implement `PATCH /v1/users/me/password` with:
- `Depends(verify_token)` — all callers authenticated
- API-key guard: if caller is service account (no `sub` claim) → 403 `ERR_API_KEY_NOT_ALLOWED`
- OIDC guard: if `user.password_hash is None` → 400 `ERR_PASSWORD_NOT_APPLICABLE`
- Password length: `8 ≤ len(new_password) ≤ 128` → 422 if violated (S003 bcrypt DoS guard)
- bcrypt verify `current_password` → 401 `ERR_WRONG_PASSWORD` if mismatch
- Hash `new_password` → UPDATE `password_hash` + `must_change_password = False`
- Audit log: `action="password_change"`, `user_id`, `timestamp`
- Return 204 No Content on success

**Review criteria**
- [ ] Route path starts with `/v1/` (R004)
- [ ] `Depends(verify_token)` present (R003)
- [ ] RBAC filter not needed (self-service: user updates their own row only)
- [ ] API-key callers → 403 (D1 from spec)
- [ ] OIDC users → 400 (D3)
- [ ] Password length validated `8–128` before bcrypt (S003)
- [ ] Audit log written (R006)
- [ ] Error shapes follow `{"error": {"code": ..., "message": ..., "request_id": ...}}` (A005)
- [ ] Test covers: success 204, wrong password 401, OIDC user 400, API-key 403, too-short password 422
