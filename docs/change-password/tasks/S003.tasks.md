### T003: Create ChangePasswordModal.tsx component

**Status**: TODO

**File(s)**
- `frontend/src/components/auth/ChangePasswordModal.tsx` — create

**Change description**
New modal component. Structure modelled on existing `UserFormModal` pattern.
Fields: Current Password, New Password, Confirm Password (all `<input type="password">`).
Behavior:
- Client-side validate before submit: new === confirm, len(new) ≥ 8 → show `.change-password-error`
- On submit: `PATCH /v1/users/me/password` with Bearer from `authStore`
- Show loading state on submit button while in-flight
- On 200/204: toast success (i18n key `auth.change_password.success`) + close modal
- On 401 `ERR_WRONG_PASSWORD`: show inline error on Current Password field
- On other error: show generic error message

**Review criteria**
- [ ] All `.change-password-*` CSS classes used (T001)
- [ ] i18n keys consumed via `useTranslation()` hook (T002)
- [ ] No inline styles (D6)
- [ ] Loading state on submit button
- [ ] `ERR_WRONG_PASSWORD` → inline field error (not toast)
- [ ] Client validation: mismatch + too-short caught before API call
- [ ] Test: render, submit success, wrong-password error, mismatch error

**Test command**
