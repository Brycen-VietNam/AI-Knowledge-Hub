## Conflicts / Gaps Found

⚠️ `AuthenticatedUser.auth_type` is `Literal["api_key", "oidc"]` — local JWT users don't fit either. **Resolution: use `"oidc"` for now** (T004 can add `"local"` if needed; out of T002 scope).

⚠️ `OAuth2PasswordRequestForm` requires `Content-Type: application/x-www-form-urlencoded`. Tests must POST form data, not JSON. Use `client.post("/v1/auth/token", data={...})` not `json={...}`.

⚠️ Rate limiter for auth endpoint needs IP-based key (not user_id — user is unknown at login time). The `RateLimiter.check()` takes `user_id: str` — pass `request.client.host` as the identifier.

⚠️ `app.state.valkey_client` is set in `create_app()` in `app.py` — auth route can access it via `request.app.state.valkey_client`. Rate limiter singleton must also be stored or instantiated at module level (same as query.py).

✅ No SQL injection risk: username lookup via `text().bindparams()`.
✅ `AUTH_SECRET_KEY` guard: if env var missing at import time → raise `RuntimeError` (same pattern as oidc.py required env var check).
