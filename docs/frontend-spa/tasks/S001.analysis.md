## Conflicts / Gaps Found

⚠️  **T002: Credentials in store** — authStore stores `username` + `password` in memory for proactive refresh. These are cleared on logout. This is acceptable per decisions (no localStorage), but the test must verify they are nulled on `logout()`.

⚠️  **T003: navigate injection** — `setNavigate(fn)` must be called AFTER Router context exists (inside `App.tsx` via `useEffect`). If `setNavigate` is called before Router mounts, navigate will be undefined when a 401 fires on the first request. Fix location: T005 App.tsx must call `setNavigate(useNavigate())` in `useEffect([], [])` at mount.

⚠️  **T004: Form encoding** — `axios.post('/v1/auth/token', data, { headers: { 'Content-Type': 'application/x-www-form-urlencoded' } })`. Must NOT use `json:` format — backend uses `OAuth2PasswordRequestForm` which parses form body. Use `URLSearchParams` or `qs.stringify`.

⚠️  **T006: i18n language persist** — `localStorage` key `"lang"` is explicitly allowed (D003) for UI language. This is the ONLY localStorage usage permitted. Token must never touch localStorage (D002). Tests should assert `localStorage.getItem("lang")` works; `localStorage.getItem("token")` must never be set.

✅  **No SQL injection risk** — frontend sends form data; all SQL parameterized in backend (S000 verified).
✅  **No hardcoded backend URL** — `import.meta.env.VITE_API_BASE_URL` is the sole source (T001 vite.config.ts).
✅  **401 handling** — Axios interceptor in T003 → `logout()` + redirect `/login`. Covers expired tokens between sessions (user returns, token stale).
✅  **Token never in localStorage** — Zustand `create()` default storage is in-memory. Do NOT pass `persist()` middleware.

### T002 — authStore
```typescript
// authStore.ts shape (no persist middleware — D002)
interface AuthState {
  token: string | null
  username: string | null
  password: string | null
  _refreshTimer: ReturnType<typeof setTimeout> | null
  login: (token: string, username: string, password: string) => void
  logout: () => void
  scheduleRefresh: (expSeconds: number, refreshFn: () => void) => void
}
```
- `scheduleRefresh`: decode `exp` from JWT payload (base64 split `token.split('.')[1]`) → `setTimeout(refreshFn, (exp - Date.now()/1000 - 300) * 1000)`
- Clear prior timer in `scheduleRefresh` before setting new one (no leak)
- `logout()` → null token + credentials + `clearTimeout(_refreshTimer)`
