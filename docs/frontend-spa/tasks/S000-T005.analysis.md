## Conflicts / Gaps Found

⚠️ `auth.py` has a module-level `RuntimeError` guard: `if not _AUTH_SECRET_KEY: raise RuntimeError(...)`.
`create_app()` in `app.py` will trigger this import. **`AUTH_SECRET_KEY` must be set in env** before `create_app()` is called. In tests, `os.environ.setdefault("AUTH_SECRET_KEY", ...)` already set at top of `test_auth.py` — smoke test is safe.

⚠️ Smoke test uses `create_app()` directly — this will attempt to connect to Valkey via `valkey_asyncio.from_url(VALKEY_URL)`. The connection is lazy (not established at construction), so no live Valkey needed for the smoke test. Rate limit branch in `auth.py` checks `if valkey_client is not None` — Valkey client will be non-None but the actual `check()` call won't fire for a single request. ✅ Safe.

⚠️ DB dependency: smoke test with bad credentials hits the DB lookup path. Must override `get_db` dependency on the real app. Use `app.dependency_overrides[get_db] = lambda: _mock_db_no_user()`.

✅ `app.openapi()["paths"]` will include `/v1/auth/token` once router is registered — review criterion satisfied automatically.
