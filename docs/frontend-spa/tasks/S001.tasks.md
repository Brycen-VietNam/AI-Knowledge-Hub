### T002: authStore (Zustand) — token + login/logout + proactive refresh

**Status**: REVIEWED

**File(s)**
- `frontend/src/store/authStore.ts` — create
- `frontend/tests/store/authStore.test.ts` — create

**Change description**
Zustand store with shape: `{ token: string|null, username: string|null, password: string|null, login(token, username, password), logout(), scheduleRefresh(expSeconds) }`.
`token` lives in memory only — NO localStorage (D002/OWASP).
`scheduleRefresh`: decode JWT `exp` from payload (base64, no verify), set `setTimeout` to re-call `/v1/auth/token` ~5 min before expiry (D011).
`logout()`: clears token + credentials + cancels any pending refresh timer.
Credentials (`username`/`password`) stored only for proactive refresh; cleared on logout.

**Review criteria**
- [ ] `token` is never written to `localStorage` or `sessionStorage`
- [ ] `scheduleRefresh` clears prior timer before setting new one (no timer leak)
- [ ] `logout()` nulls credentials AND token
- [ ] Tests: login sets token, logout clears token, scheduleRefresh fires callback before exp

**Test command**
```bash
cd frontend && npm run test -- src/store/authStore.test.ts
```

**Parallel**: after:T001
**Size estimate**: ~45 lines (store) + ~40 lines (tests)
