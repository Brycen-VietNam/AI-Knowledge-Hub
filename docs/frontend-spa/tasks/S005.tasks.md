### T001: Create frontend/Dockerfile (multi-stage build)

**Status**: DONE

**File(s)**
- `frontend/Dockerfile` — create

**Change description**
Multi-stage Docker build: stage 1 uses `node:20-alpine` to install deps and build the Vite
bundle; stage 2 uses `nginx:alpine` to serve the static output. `npm ci` (not `npm install`)
for reproducible installs. Built dist goes to `/usr/share/nginx/html`.

**Key changes**
```dockerfile
# Stage 1 — build
FROM node:20-alpine AS build
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

# Stage 2 — serve
FROM nginx:alpine
COPY --from=build /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

### T002: Create frontend/nginx.conf (SPA fallback routing)

**Status**: DONE

**File(s)**
- `frontend/nginx.conf` — create

**Change description**
nginx server block for serving a React SPA: all unmatched routes fall back to `index.html`
so React Router handles client-side navigation. Gzip compression enabled. Static asset
caching for JS/CSS files.

**Key changes**
```nginx
server {
    listen 80;
    root /usr/share/nginx/html;
    index index.html;

    # SPA fallback — all routes handled by React Router
    location / {
        try_files $uri $uri/ /index.html;
    }

    # Cache static assets (Vite generates hashed filenames)
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff2?)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # Security headers
    add_header X-Frame-Options "SAMEORIGIN";
    add_header X-Content-Type-Options "nosniff";
}
```

**Review criteria**
- [ ] `listen 80;` at top of server block
- [ ] `try_files $uri $uri/ /index.html;` for SPA routing
- [ ] Static asset cache headers present
- [ ] `X-Frame-Options: SAMEORIGIN` header set
- [ ] `X-Content-Type-Options: nosniff` header set
- [ ] No hardcoded backend URLs in nginx.conf

### T004: Add frontend-spa service to docker-compose.yml

**Status**: DONE

**File(s)**
- `docker-compose.yml` — modify: add `frontend-spa` service block

**Change description**
Add a `frontend-spa` service that builds from `./frontend`, maps port `8080:80`, passes
`VITE_API_BASE_URL` as a build arg (not runtime env — Vite bakes env vars at build time),
and depends on the `app` service being started.

**Important**: Vite bakes `VITE_*` vars at **build time**, not runtime. The
`environment:` block in docker-compose has no effect on Vite bundles. Add `args:` under
`build:` so `VITE_API_BASE_URL` is passed to `docker build --build-arg`.

**Key changes**
```yaml
  frontend-spa:
    build:
      context: ./frontend
      args:
        VITE_API_BASE_URL: ${VITE_API_BASE_URL:-http://localhost:8000}
    container_name: knowledge-hub-frontend
    restart: unless-stopped
    ports:
      - "8080:80"
    depends_on:
      - app
```

Also update `frontend/Dockerfile` stage 1 to accept the build arg:
```dockerfile
# After: FROM node:20-alpine AS build
ARG VITE_API_BASE_URL=http://localhost:8000
ENV VITE_API_BASE_URL=$VITE_API_BASE_URL
```

**Review criteria**
