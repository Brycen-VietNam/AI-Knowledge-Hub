**Password hash column sizing:** `VARCHAR(255)` is correct for bcrypt hashes — bcrypt output is 60 chars; 255 gives headroom for algorithm prefixes (e.g. `$2b$12$...`). Alternatively, plain `TEXT` is also acceptable (PostgreSQL stores both identically). Use `TEXT` for consistency with `email` and `display_name` columns in same table.

-- Nullable: OIDC users have no local password (NULL is correct, not an error).

**Why `IF EXISTS` in rollback:** 007 uses plain `DROP COLUMN` without `IF EXISTS`, but adding `IF EXISTS` to the rollback comment makes it idempotent — safer for re-run in CI. Not a requirement, just recommended.
